  \subsection{Déclaration d'une classe}
  \subsubsection{Analyse syntaxique}
	
si le mot clé T\_CLASS est trouvé alors this->isclass = true, on recherche un identifant dans this->classes, s'il n'est pas trouvé on crée un S\_CLASS lexical->getcurrentlexeme(), this->nbclasses et on l'insère dans classes.
On incrémente this->nbclasses et this->current_classe = classe
On crée un nouveau map class_variables de d'identifiants
On pousse class_variables dans chained_variables   
Et this->variables = class_variables;

puis on execute this->bloc_classe(node, node);

puis this->isclass = false;
et this->variables = this->chained_variables[0];

puis on crée un S_CLASS_END

case S_CLASS transformé en TAC_ASSIGN_CLASS, node->getvertex()->getaddr(), node->getvertex()->getnblocalvariables(), node->getvertex()->getlexeme()
et S_CLASS_END transformé en TAC_ASSIGN_ENDCLASS, node->getid(), node->getid()

  \subsection{Déclaration d'une variable}
  \subsubsection{Analyse syntaxique}
L'analyse syntaxique est principalement implémentée dans la classe \textbf{compiler/syntax/t\_syntax.cpp}

Les compteurs nbexprstermsfactors et nbexprstermsfactors\_prog sont incrémentés de 1

Si le mot clé declr est trouvé puis une variable \$identifiant on vérifie si cette variable est déclarée dans une classe (isclass = true), si c'est le cas alors on vérifie que son nom n'est pas déjà pris dans le vecteur des variables de la classe courante (current\_classe->getvariables()) et on l'ajoute (current\_classe->getvariables()->insert()). On crée alors un t\_identifier de type S\_DECLARATION avec comme identifiant numérique current\_classe->getvariables()->size(). Dans le cas ou on est pas dans la déclaration d'une variable de classe, même chose mais le vecteur variables est utilisé en remplacement de current\_classe.\\\\
On crée ensuite un t\_identifier de type S\_VARIABLE puis S\_IDENTIFIER puis S\_DOTCOMMA


  \subsubsection{Transformation en Three Address Code}

Seul le t\_identifier de type S\_DECLARATION entre en jeu dans le construction du three address code TAC\_ASSIGN\_DECLR avec comme argurment node->getvertex()->getaddr() qui est la valeur de current\_classe->getvariables()->size() ou variables->size()


  \subsubsection{Transformation en Opcodes}

Aucun code généré


\subsection{Affectation d'une variable}
  \subsubsection{Analyse syntaxique}

On crée un S\_AFFECTATION avec comme argument identifier\_found->getaddr() puis un S\_EQUAL

  \subsubsection{Transformation en Opcodes}

EMPL (*it).getaddrarg1() (exemple : 3)\\

\subsection{Utilisation d'une variable}
  \subsubsection{Analyse syntaxique}

On crée un map variables\_temp qui pointe sur le map de variables globales (this->variables).
Si la variable est une classe alors variables\_temp pointe sur les variables de classe (identifier\_class\_found->getclass()->getvariables()). 
Si on trouve this alors variables\_temp pointe sur les variables de la classe courante (this->current\_classe->getvariables())
Si une variable est trouvé dans variables\_temp alors on crée un t\_symbol de type S\_VARIABLE avec comme argument identifier\_found->getaddr()

  \subsubsection{Transformation en Three Address Code}

S\_VARIABLE est transformé en TAC\_ASSIGNV, node->getvertex()->getaddr(), node->getchild(0)->getvertex()->getaddr()));
ou node->getvertex()->getaddr() est l'adresse de S\_FACTOR soit this->variables->size() et node->getchild(0)->getvertex()->getaddr() l'adresse de S\_VARIABLE soit identifier\_found->getaddr()

  \subsubsection{Transformation en Opcodes}

EMPL\\
(*it).getaddrarg1()\\
DEPL\\
(*it).getaddrresult()\\
